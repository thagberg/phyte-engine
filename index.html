Create custom fighting games without dedicated programmer help

Phyte Engine includes a dynamic entity-component based fighting game engine runtime and a toolkit which allows game designers to define entities, components, rules, and states without needing to write a lot of code.

Entity-Component architecture

An entity-component based runtime architecture enables a great deal of flexibility in the kinds of games which can be designed. This means that the Phyte Engine system supports traditional 1vs1 2D "street brawling" style fighting games, or even more frantic party-game styled arena fighting games.

Pipelined toolkit process

The toolkit supports a pipelined process, which allows designers to string together dependent entities and components with ease. The runtime will recognize the relationships between components for use in the game.

Permissive Open Source License

An MIT license is applied to the Phyte Engine runtime and toolkit code, which means it can be used for free or commercial projects.

Installation Dependencies

The engine runtime depends on the PyGame open source library, and the toolkit depends on Qt 3.
